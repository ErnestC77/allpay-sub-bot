# Telegram-бот продажи подписок (AllPay)

Бот продаёт подписку внутри Telegram, оплата — через израильский шлюз **AllPay** (валюта ILS).
Стек: **Python + aiogram 3**, **PostgreSQL 16** (SQLAlchemy async / asyncpg), aiohttp-webhook.

## Возможности
- `/start` → выбор языка (14 языков, i18n; полностью переведены RU и EN).
- Приветствие с картинкой и кнопкой «Купить подписку».
- Выбор срока подписки → карточка тарифа (фото + описание) → выбор способа оплаты.
- Обязательный ввод e-mail перед оплатой (если не указан в профиле).
- Создание счёта в AllPay и подтверждение оплаты по webhook (идемпотентно).
- Главное меню: «Тарифные планы», «Моя подписка» (+ «Продлить»), «Мой аккаунт»
  (язык / часовой пояс / почта), «Поддержка».
- Админ-панель в боте (`/admin`, только для `ADMIN_IDS`): правка цены, срока, названия,
  описания, фото и видимости тарифов.
- Автоуведомления об окончании подписки: несколько настраиваемых порогов (за N дней),
  у каждого свой текст; шлётся один раз на подписку. Настраивается в `/admin → Уведомления`.

## Структура
```
bot.py            — точка входа: polling + aiohttp-webhook в одном процессе
config.py         — конфиг из .env
i18n.py, locales/ — переводы
db/               — модели, движок, CRUD (User / Plan / Subscription / Payment)
handlers/         — start, menu, plans, purchase, subscription, account, support, admin
keyboards/        — inline и reply клавиатуры
payments/         — allpay.py (подпись, создание платежа), webhook.py (приём уведомлений)
scheduler.py      — фоновая рассылка уведомлений об окончании подписки
utils.py          — валидация e-mail/часового пояса, форматирование дат
```

## Запуск локально
1. `python -m venv .venv && .venv\Scripts\activate`
2. `pip install -r requirements.txt`
3. Создать БД PostgreSQL 16 и скопировать `.env.example` → `.env`, заполнить:
   - `BOT_TOKEN`, `ADMIN_IDS`, `SUPPORT_CONTACT`
   - `ALLPAY_LOGIN`, `ALLPAY_KEY` (из ЛК AllPay: Settings → Integrations)
   - `DATABASE_URL` (postgresql+asyncpg://...)
   - `WEBHOOK_BASE_URL` — публичный URL для webhook AllPay. Локально — туннель
     (ngrok/cloudflared), например `https://xxxx.ngrok.io`.
4. (Опционально) положить приветственную картинку в `assets/welcome.jpg`.
5. `python bot.py`

Тарифы по умолчанию создаются при первом запуске (30/60/90 дней) — цены и тексты
затем правятся через `/admin`. Картинки тарифов загружаются там же (хранятся как Telegram `file_id`).

## Webhook AllPay
- Адрес приёма уведомлений: `{WEBHOOK_BASE_URL}/allpay/webhook` (POST).
- Подпись проверяется SHA256 по алгоритму AllPay; при `status=1` подписка активируется/продлевается.
- Обработчик идемпотентен (AllPay ретраит до 10 раз) — повторные уведомления игнорируются.
- Проверка здоровья: `GET /health`.

> Точный порядок формирования подписи для вложенных `items` желательно сверить с
> официальным PHP-примером из ЛК AllPay (см. комментарий в `payments/allpay.py`).

## Уведомления об окончании подписки
- Пороги (за сколько дней предупреждать) настраиваются в `/admin → Уведомления`.
  По умолчанию засеян один порог — 14 дней. Можно добавлять несколько (14, 7, 1 …),
  включать/выключать и задавать свой текст для каждого.
- В тексте порога доступны подстановки `{days}` (дней до конца) и `{date}` (дата окончания).
- Фоновая задача (`scheduler.py`) проверяет подписки каждые `REMINDER_CHECK_INTERVAL`
  секунд (по умолчанию раз в час). Каждый порог отправляется один раз на подписку
  (дедуп через таблицу `reminder_logs`), поэтому частые проверки безопасны.
- Продление создаёт новую подписку — пороги отрабатывают для неё заново.

## Деплой (Render)
Сервис типа **Web Service** (aiohttp биндит `PORT`). Переменные окружения — как в `.env`.
`WEBHOOK_BASE_URL` = публичный URL сервиса.
